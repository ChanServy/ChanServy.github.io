链路追踪

2020年12月25日
| Spring Cloud
| 阅读
文章目录

前言：在这种大规模的分布式系统中，一个完整的系统是由多个不同的服务来共同支撑的。不同的系统可能分布在上千台服务器上，横跨多个不同的数据中心。一旦系统出问题，此时问题的定位就比较麻烦。链路追踪可以解决。

链路追踪

一、Spring Cloud Sleuth

1、简介

分布式链路追踪：

在微服务环境下，一次客户端请求，可能会引起数十次、上百次服务端服务之间的调用。一旦请求出问题，我们需要考虑到很多东西：

  • 如何快速定位问题？
  • 如何快速确定此次客户端调用都涉及到哪些服务？
  • 到底是哪一个服务出问题了？

要解决这些问题，就涉及到分布式链路追踪。

分布式链路追踪系统主要是用来跟踪服务调用记录的，一般来说，一个分布式链路追踪系统，有三个部分：

  • 数据收集
  • 数据存储
  • 数据展示

Spring Cloud Sleuth 是 Spring Cloud 提供的一套分布式链路追踪系统。

trace：从请求到达系统开始，到给请求作出响应，这样一个过程称为 trace 。

span：每次调用服务时，埋入的一个调用记录，称为 span 。

annotation：相当于 span 的语法，描述 span 所处的状态。

2、简单应用

首先创建一个项目，引入 Spring Cloud Sleuth

<dependencies>
    <dependency>
        <groupId>org.springframework.boot</groupId>
        <artifactId>spring-boot-starter-web</artifactId>
    </dependency>
    <dependency>
        <groupId>org.springframework.cloud</groupId>
        <artifactId>spring-cloud-starter-sleuth</artifactId>
    </dependency>
</dependencies>

接下来创建一个 HelloController ，打印日志测试：

/**
 * @author chan
 * @version V1.0
 * @Package ink.csy.sleuth.controller
 * @date 2020/10/24 下午7:40
 * @Description
 */
@RestController
public class HelloController {

    private static final Log log = LogFactory.getLog(HelloController.class);

    @Autowired
    RestTemplate restTemplate;

    @Autowired
    HelloService helloService;

    @GetMapping("/hello")
    public String hello(){
        log.info("hello spring cloud sleuth.");
        return "hello spring cloud sleuth.";
    }
}

给当前的服务配置一个名字，这个名字会在输出的日志中体现出来：

spring.application.name=CSY-sleuth
server.port=10001

启动应用，请求 hello 接口，结果如下：

2020-12-25 15:19:56.332  INFO [CSY-sleuth,eb33d0910242cb02,eb33d0910242cb02,false] 6988 --- [io-10001-exec-9] i.csy.sleuth.controller.HelloController  : hello spring cloud sleuth.

这个就是 Spring Cloud Sleuth 的输出。

再定义两个接口，在 hello2 中调用 hello3 ，形成调用链：

@GetMapping("/hello2")
public String hello2() throws InterruptedException {
    log.info("hello2");
    Thread.sleep(500);
    return restTemplate.getForObject("http://localhost:10001/hello3",String.class);
}

@GetMapping("/hello3")
public String hello3() throws InterruptedException {
    log.info("hello3");
    Thread.sleep(500);
    return "hello3";
}

此时，访问 hello2 ，会先调用 hello3 ，拿到返回的结果，会给 hello2 。

一个 trace 由多个 span 组成，一个 trace 相当于一个调用链，而一个 span 则是这个链中的每一次调用过程。

Spring Cloud Sleuth 中也可以收集到异步任务中的信息。

开启异步任务：

/**
 * @author chan
 * @version V1.0
 * @Package ink.csy.sleuth
 * @date 2020/10/24 下午7:36
 * @Description
 */
@SpringBootApplication
@EnableAsync//开启异步任务
public class SleuthApplication {
    public static void main(String[] args) {
        SpringApplication.run(SleuthApplication.class,args);
    }

    /**
     * 不涉及ribbon和feign,直接通过rest调用
     * @return
     */
    @Bean
    RestTemplate restTemplate(){
        return new RestTemplate();
    }
}

/**
 * @author chan
 * @version V1.0
 * @Package ink.csy.sleuth.service
 * @date 2020/10/24 下午8:38
 * @Description
 */
@Service
public class HelloService {
    private static final Log LOGGER = LogFactory.getLog(HelloService.class);

    @Async
    public String asyncBackgroundFun(){
        LOGGER.info("asyncBackgroundFun");
        return "asyncBackgroundFun";
    }
}

再在 controller 中调用该异步方法：

@GetMapping("/hello4")
public String hello4(){
    log.info("hello4");
    return helloService.asyncBackgroundFun();
}

启动项目，访问 hello4 进行进行测试，发现 Sleuth 也打印日志了，在异步任务中，异步任务都是单独的 spanID。

Spring Cloud Sleuth 也可以收集定时任务的信息。

首先开启定时任务支持：

@SpringBootApplication
@EnableAsync//开启异步任务
@EnableScheduling//开启定时任务支持
public class SleuthApplication {
    public static void main(String[] args) {
        SpringApplication.run(SleuthApplication.class,args);
    }

    /**
     * 不涉及ribbon和feign,直接通过rest调用
     * @return
     */
    @Bean
    RestTemplate restTemplate(){
        return new RestTemplate();
    }
}

然后在 service 中，添加定时任务，去调用刚刚的 asyncBackgroundFun() 方法：

@Scheduled(cron = "0/10 * * * * ?")
public void sche1(){
    LOGGER.info("start:");
    asyncBackgroundFun();
    LOGGER.info("end");
}

开启这个服务，因为是定时任务，不用访问接口，根据定时的时间，它自己就会在控制台输出。在定时任务中，每一轮定时任务都会产生一个新的 Trace ，并且在每轮的调用过程中，SpanID 都是一致的，这个和普通的调用不一样。
