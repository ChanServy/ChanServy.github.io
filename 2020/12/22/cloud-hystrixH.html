前言：Spring Cloud之Hystrix(H版)。

一、断路器简介

在微服务架构中，根据业务来拆分成一个个的服务，服务与服务之间可以相互调用（RPC），在Spring Cloud可以用RestTemplate+Ribbon和Feign来调用。为了保证其高可用，单个服务通常会集群部署。由于网络原因或者自身的原因，服务并不能保证100%可用，如果单个服务出现问题，调用这个服务就会出现线程阻塞，此时若有大量的请求涌入，Servlet容器的线程资源会被消耗完毕，导致服务瘫痪。服务与服务之间的依赖性，故障会传播，会对整个微服务系统造成灾难性的严重后果，这就是服务故障的“雪崩”效应。

为了解决这个问题，业界提出了断路器模型。

Netflix has created a library called Hystrix that implements the circuit breaker pattern. In a microservice architecture it is common to have multiple layers of service calls. —-摘自官网

Netflix开源了Hystrix组件，实现了断路器模式，SpringCloud对这一组件进行了整合。 在微服务架构中，一个请求需要调用多个服务是非常常见的，如下图：

较底层的服务如果出现故障，会导致连锁故障。当对特定的服务的调用的不可用达到一个阀值（Hystric 是5秒20次） 断路器将会被打开。

断路打开后，可用避免连锁故障，fallback方法可以直接返回一个固定值。

二、准备工作

基于前几篇的文章，我们先启动 CSY-eureka ，然后启动一个 CSY-provider ，它的端口号是20001。

三、在ribbon使用断路器

将之前的 CSY-ribbon 进行改造，首先在原来的基础上添加hystrix的依赖，也就是spring-cloud-starter-netflix-hystrix：

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-hystrix</artifactId>
</dependency>

在程序的启动类RibbonApplication 加@EnableHystrix注解开启Hystrix：

/**
 * @Classname RibbonApplication
 * @Description TODO
 * @Date 2020/9/13 21:55
 * @Created by ChenSiYu
 */
//@EnableDiscoveryClient //@EnableEurekaClient差不多 不用Eureka做注册中心的时候用这个注解
//@RibbonClients({@RibbonClient("CSY-PROVIDER1"),@RibbonClient("CSY-PROVIDER2")})//指定多个服务开启负载均衡
@EnableEurekaClient
@SpringBootApplication
//开启负载均衡,参数为要给哪个服务开启,被开启的服务在访问的时候会有负载均衡,并且设置负载均衡的策略为随机
@RibbonClient(value = "CSY-PROVIDER",configuration = RibbonConfig.class)
@EnableHystrix//@EnableHystrix继承了@EnableCircuitBreaker,表示开启断路器,当我们的调用出现问题或者是流量过高的时候就会触发降级++
public class RibbonApplication {

    public static void main(String[] args) {
        SpringApplication.run(RibbonApplication.class, args);
    }



    @Bean
    @LoadBalanced//给rest模板开启负载均衡,一定要加这个注解,不然的话会提示找不到主机
    RestTemplate restTemplate(){
        return new RestTemplate();
    }

}

在调用服务的方法上加上@HystrixCommand注解。该注解对该方法创建了熔断器的功能，并指定了fallbackMethod熔断方法。另外新建对应的熔断方法。

/**
 * @Classname OrderController
 * @Description TODO
 * @Date 2020/9/14 19:31
 * @Created by ChenSiYu
 */
@RestController
public class OrderController {

    @Autowired
    private RestTemplate restTemplate; //一个专门请求rest风格url的对象

    @Autowired
    private LoadBalancerClient loadBalancerClient;//负载均衡的客户端

    @Autowired
    OrderService orderService;

    @RequestMapping("/order/{id}")
    @HystrixCommand(fallbackMethod = "getUserInfoHystrix",
            commandProperties = {//模拟非常短的超时时间,我们的每次请求都凉了,当然我们也可以关掉相应的服务来测试
                    @HystrixProperty(name = "execution.isolation.strategy", value = "THREAD"),
                    @HystrixProperty(name = "execution.isolation.thread.timeoutInMilliseconds", value = "1")})
    public TbUser getUserInfo(@PathVariable long id) {
        TbUser user = restTemplate.getForObject("http://CSY-PROVIDER/user/findUser/" + id, TbUser.class);
        return user;
    }

    public TbUser getUserInfoHystrix(long id) {
        TbUser tbUser = new TbUser();
        tbUser.setUserId(-1);
        tbUser.setUsername("妖怪");
        tbUser.setPassword("250250");
        tbUser.setGender("无");
        tbUser.setAge(250);
        return tbUser;
    }

    @RequestMapping("/test")
    @HystrixCommand(fallbackMethod = "chooseHystrix")
    public String choose() {
        int port = loadBalancerClient.choose("CSY-PROVIDER").getPort();
        System.err.println("----------------->" + port);
        return port + "";
    }

    public String chooseHystrix() {
        return "在等端口号么?小鬼,别天真了!";
    }

    @RequestMapping("/hi/{name}")
    public String hi(@PathVariable String name) {
        String servicePort = orderService.getServicePort(name);
        return servicePort;
    }
}

/**
 * @Classname OrderService
 * @Description TODO
 * @Date 2020/9/15 12:30
 * @Created by ChenSiYu
 */
@Service
public class OrderService {
    @Autowired
    RestTemplate restTemplate;

    @HystrixCommand(fallbackMethod = "getServicePortHystrix",
            commandProperties = {
                    @HystrixProperty(name = "execution.isolation.strategy", value = "THREAD"),
                    @HystrixProperty(name = "execution.isolation.thread.timeoutInMilliseconds", value = "1")})
    public String getServicePort(String name) {
        return restTemplate.getForObject("http://CSY-PROVIDER/hi?name=" + name, String.class);
    }

    public String getServicePortHystrix(String name) {
        return "别等了,你等不到它了...";
    }
}

上面我们有两个方法是模拟非常短的超时时间，假设我们的每次请求都凉了，当然我们也可以关掉相应的服务来测试。

对比 Spring Cloud之Ribbon(H版) 文章中的测试结果截图，我们发现我们实现了降级熔断的效果。

四、在Feign中使用断路器

Feign是自带断路器的，在D版本之后的 Spring Cloud 中，它没有默认打开。需要在配置文件中配置打开它，在配置文件加以下代码：

feign.hystrix.enabled=true

做好准备工作，将 CSY-eureka 打开，然后分别打开 CSY-feign，和 CSY-feign-config 模块，不开启提供者服务模块，模拟提供者服务宕机的效果，并且与 Spring Cloud之Feign(H版) 中的测试结果截图对比。

对于feign的熔断降级方式有两种，分别为接口的方式，和工厂的方式。我们正好有 CSY-feign （port：20004）和 CSY-feign-config （port：20005）两个服务可以分别用来测试这两种熔断降级的方式。

1、对CSY-feign进行改造

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-hystrix</artifactId>
</dependency>

@SpringBootApplication
@EnableEurekaClient
@EnableFeignClients
@EnableHystrix
public class FeignApplication {
    public static void main(String[] args) {
        SpringApplication.run(FeignApplication.class, args);
    }
}

server.port=20004

spring.application.name=CSY-feign

eureka.client.serviceUrl.defaultZone=http://localhost:20000/eureka/

#必须开启这个,否无断路器无效
feign.hystrix.enabled=true

新建 OrderFeignFallback 类实现 OrderFeign 接口，@Component必须注入到Ioc容器中。

/**
 * @Classname OrderFeignFallback
 * @Description TODO
 * @Date 2020/9/17 23:02
 * @Created by ChenSiYu
 */
@Component
public class OrderFeignFallback implements OrderFeign {
    @Override
    public List<TbUser> findUserList() {
        TbUser tbUser = new TbUser();
        tbUser.setUserId(-1);
        tbUser.setUsername("妖怪");
        tbUser.setPassword("250250");
        tbUser.setGender("无");
        tbUser.setAge(250);
        ArrayList<TbUser> tbUsers = new ArrayList<>(5);
        for (int i = 0; i < 5; i++) {
            tbUsers.add(tbUser);
        }
        return tbUsers;
    }

    @Override
    public TbUser findUser(long userId) {
        TbUser tbUser = new TbUser();
        tbUser.setUserId(-1);
        tbUser.setUsername("妖怪");
        tbUser.setPassword("250250");
        tbUser.setGender("无");
        tbUser.setAge(250);
        return tbUser;
    }

    @Override
    public String home(String name) {
        return "莫道征途路漫漫...";
    }
}

/**
 * @Classname OrderFeign
 * @Description TODO
 * @Date 2020/9/15 22:13
 * @Created by ChenSiYu
 */
//配置当前接口为Feign的客户端,内部访问CSY-provider这个服务集群,并且指定失败后执行OrderFeignFallback中对应的方法
@FeignClient(value = "CSY-PROVIDER",fallback = OrderFeignFallback.class)
public interface OrderFeign {

    @RequestMapping("/user/findUserList")
    List<TbUser> findUserList();

    @GetMapping("/user/findUser/{userId}")
    TbUser findUser(@PathVariable("userId") long userId);//注意在feign中PathVariable注解必须指定参数的名字,不指定会报错

    @RequestMapping("/hi")
    String home(@RequestParam(value = "name", defaultValue = "erdochan") String name);
}

其他的在原基础上不用动，重新启动 CSY-feign 服务，开始测试。

/**
 * @Classname OrderFeignFallbackFactory
 * @Description TODO
 * @Date 2020/9/17 23:30
 * @Created by ChenSiYu
 */
//泛型,当前的工厂是给哪个feign client用的,就写哪个
@Component
public class OrderFeignFallbackFactory implements FallbackFactory<OrderFeign> {

    @Override
    public OrderFeign create(Throwable throwable) {
        //匿名内部类
        return new OrderFeign() {
            @Override
            public List<TbUser> findUserList() {
                TbUser tbUser = new TbUser();
                tbUser.setUserId(-1);
                tbUser.setUsername("妖怪");
                tbUser.setPassword("250250");
                tbUser.setGender("无");
                tbUser.setAge(250);
                ArrayList<TbUser> users = new ArrayList<>();
                for (int i = 0; i < 5; i++) {
                    users.add(tbUser);
                }
                return users;
            }

            @Override
            public TbUser findUser(long userId) {
                TbUser tbUser = new TbUser();
                tbUser.setUserId(-1);
                tbUser.setUsername("妖怪");
                tbUser.setPassword("250250");
                tbUser.setGender("无");
                tbUser.setAge(250);
                return tbUser;
            }
        };
    }
}
