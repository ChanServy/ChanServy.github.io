1. Spring Boot自带的JSON

2. 创建User

/**
 * @Classname User
 * @Description TODO
 * @Date 2020/5/18 13:36
 * @Created by ChenSiYu
 */
@Data
public class User {
    private String name;
    private String address;
    private String tel;
    @JsonFormat(pattern = "yyyy-MM-dd")
    private Date birthday;
}

3. 创建UserController

/**
 * @Classname UserController
 * @Description TODO
 * @Date 2020/5/18 13:39
 * @Created by ChenSiYu
 */
@RestController
public class UserController {

    @GetMapping("/json")
    public List<User> getUserInfo() {
        List<User> users = new ArrayList<>();
        for (int i = 0; i < 5; i++) {
            User user = new User();
            user.setName("chen======>" + i);
            user.setAddress("anshan======>" + i);
            user.setTel("666666======>" + i);
            user.setBirthday(new Date());
            users.add(user);
        }
        return users;
    }
}

4. 运行结果

上面的案例中，我们发现在Spring Boot中什么有关Json的依赖都不需要导入，直接导入了spring-boot-starter-web依赖就可以在Spring Boot中通过@RestController注解将服务端返回的对象序列化成 JSON 字符串了，为什么呢？因为Spring Boot中的就集成了Jackson，如下图：

这就是为什么在刚刚的例子中可以直接将对象转化为Json，但是它的底层是什么样的呢？HttpMessageConverter ！看名字就知道，这是一个消息转换工具，有两方面的功能：

  • 将服务端返回的对象序列化成 JSON 字符串

  • 将前端传来的 JSON 字符串反序列化成 Java 对象

所有的 JSON 生成都离不开相关的 HttpMessageConverter

SpringMVC 自动配置了 Jackson 和 Gson 的 HttpMessageConverter，Spring Boot 中又对此做了自动化配置：

  1. org.springframework.boot.autoconfigure.http.JacksonHttpMessageConvertersConfiguration
  2. org.springframework.boot.autoconfigure.http.GsonHttpMessageConvertersConfiguration

所以，如果用户使用 jackson 和 gson 的话，没有其他额外配置，则只需要添加依赖即可。

上述的对象中对于日期的格式化使用了@JsonFormat注解，这是一个简易的demo，但是如果项目的工程量大了的话，对象多了，需要格式化的更多，一个一个写就比较麻烦了，这个时候我们可以定义一个全局的对于日期的格式化，怎么定义呢？需要从底层入手，有关Jackson的，大家知道在Jackson中做序列化的工具就是ObjectMapper，如下：

/**
 * @Classname User
 * @Description TODO
 * @Date 2020/5/18 13:36
 * @Created by ChenSiYu
 */
@Data
public class User {
    private String name;
    private String address;
    private String tel;
//    @JsonFormat(pattern = "yyyy-MM-dd")
    private Date birthday;
}

将注解注释掉，然后新建配置类：

/**
 * @Classname DateConfig
 * @Description 日期格式化配置类
 * @Date 2020/5/18 18:14
 * @Created by ChenSiYu
 */
@Configuration
public class DateConfig {

    @Bean
    ObjectMapper objectMapper(){
        ObjectMapper objectMapper = new ObjectMapper();
        objectMapper.setDateFormat(new SimpleDateFormat("yyyy/MM/dd"));
        return objectMapper;
    }

}

2. FastJson

与Jackson不一样，在Spring Boot的里面并没有集成FastJson的HttpMessageConverter，FastJson是阿里巴巴开源的对于Json的解析工具。上面的案例中，加了web的依赖后，直接可以用Jackson，至于后来的配置类，只不过是为了改善日期的显示，但是这里的FastJson，再加入了依赖后，还必须需要添加配置类，这里不添加配置类是不行的。

①在原基础上添加依赖

<dependency>
    <groupId>com.alibaba</groupId>
    <artifactId>fastjson</artifactId>
    <version>1.2.60</version>
</dependency>

②配置类

    /**
     * @Description 阿里巴巴的json
     * @Param []
     * @return com.alibaba.fastjson.support.spring.FastJsonHttpMessageConverter
     * @Date 2020/5/19 17:51
     * @Author ChenSiYu
     */
    @Bean
    FastJsonHttpMessageConverter fastJsonHttpMessageConverter(){
        FastJsonHttpMessageConverter fastJsonHttpMessageConverter = new FastJsonHttpMessageConverter();
        FastJsonConfig fastJsonConfig = new FastJsonConfig();
        fastJsonConfig.setDateFormat("yyyy-MM-dd");
        fastJsonHttpMessageConverter.setFastJsonConfig(fastJsonConfig);
        return fastJsonHttpMessageConverter;
    }
