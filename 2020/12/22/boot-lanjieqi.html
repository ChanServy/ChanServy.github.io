3、创建拦截器

实现HandlerInterceptor接口中的方法，ctrl+O，查看接口中的方法。

preHandle方法是进行处理器拦截用的，顾名思义，该方法将在Controller处理之前进行调用。

postHandle是进行处理器拦截用的，它的执行时间是在处理器进行处理之后，也就是在Controller的方法调用之后执行，但是它会在DispatcherServlet进行视图的渲染之前执行，也就是说在这个方法中你可以对ModelAndView进行操作。

afterCompletion该方法将在整个请求完成之后，也就是DispatcherServlet渲染了视图之后执行。（这个方法的主要作用是用于清理资源的）

/**
 * @Classname MyInterceptor
 * @Description TODO
 * @Date 2020/5/28 18:05
 * @Created by ChenSiYu
 */
public class MyInterceptor implements HandlerInterceptor {
    /**
     * @Description 重写preHandle方法，在请求发生前执行。
     * @Param [request, response, handler] 
     * @return boolean
     * @Date 2020/5/29 22:32
     * @Author ChenSiYu
     */
    @Override
    public boolean preHandle(HttpServletRequest request, HttpServletResponse response, Object handler) throws Exception {
        System.out.println("preHandle");
        return true;
    }

    /**
     * @Description 重写postHandle方法，在请求完成后执行。
     * @Param [request, response, handler, modelAndView] 
     * @return void
     * @Date 2020/5/29 22:32
     * @Author ChenSiYu
     */
    @Override
    public void postHandle(HttpServletRequest request, HttpServletResponse response, Object handler, ModelAndView modelAndView) throws Exception {
        System.out.println("postHandle");
    }

    /**
     * @Description 该方法将在整个请求完成之后，也就是DispatcherServlet渲染了视图之后执行。
     * @Param [request, response, handler, ex]
     * @return void
     * @Date 2020/5/29 22:33
     * @Author ChenSiYu
     */
    @Override
    public void afterCompletion(HttpServletRequest request, HttpServletResponse response, Object handler, Exception ex) throws Exception {
        System.out.println("afterCompletion");
    }
}

        registry.addInterceptor(myInterceptor()).addPathPatterns("/**");
