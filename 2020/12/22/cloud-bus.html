Spring Cloud Bus消息总线

2020年12月22日
| Spring Cloud
| 阅读
文章目录

前言：Spring Cloud Bus消息总线。

Spring Cloud Bus

Spring Cloud Bus通过轻量级的消息代理链接各个微服务，可以用来广播配置文件的更改，或者管理服务监控。

docker中安装rabbitmq：

#指定版本，该版本包含了web控制页面
docker pull rabbitmq:management
#方式一：默认guest 用户，密码也是 guest
docker run -d --hostname my-rabbit --name rabbit -p 15672:15672 -p 5672:5672 rabbitmq:management

#方式二：设置用户名和密码
docker run -d --hostname my-rabbit --name rabbit -e RABBITMQ_DEFAULT_USER=user -e RABBITMQ_DEFAULT_PASS=password -p 15672:15672 -p 5672:5672 rabbitmq:management

http://localhost:15672/

首先给config-server和config-client分别加上Spring Cloud Bus依赖：

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-bus-amqp</artifactId>
</dependency>

然后，给两个分别配置mq，使之都链接到rabbitmq上：

# 链接消息队列
spring.rabbitmq.host=localhost
spring.rabbitmq.port=5672
spring.rabbitmq.username=guest
spring.rabbitmq.password=guest

同时，由于这次由config-server提供刷新接口，所以给config-server加上actuator依赖：

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-actuator</artifactId>
</dependency>

然后，记得一定要在config-server中，添加开启bus-refresh端点：

# 开启bus-refresh端点
management.endpoints.web.exposure.include=bus-refresh

由于前面给config-server中的所有的接口都添加了保护，所以刷新接口将无法直接访问，此时，可以通过修改security的配置，对端点的权限作出修改：

/**
 * @author chan
 * @version V1.0
 * @Package ink.csy.server.config
 * @date 2020/10/18 下午6:38
 * @Description
 */
@Configuration
public class SecurityConfig extends WebSecurityConfigurerAdapter {

    @Override
    protected void configure(HttpSecurity http) throws Exception {
        http.authorizeRequests()
                .anyRequest().authenticated()
                .and()
                .httpBasic()
                .and()
                .csrf().disable();
    }
}

这段配置中，开启了HttpBasic登录，这样，在发送刷新请求的时候，就可以直接通过HttpBasic配置认证信息了。

最后分别启动config-server和config-client，然后修改配置信息并提交到github，刷新config-client的接口看是否有变化（此时无变化）。然后，使用postman的POST请求，并且选中postman认证类型中的Basic Auth填入用户名密码（在配置文件中自己设置的那个），http://localhost:8081/actuator/bus-refresh。

这次的post是针对config-server的，config-server会把这个刷新的指令传递到消息队列中，然后rabbitmq再把这个指令传递给各个client。刷新config-client的接口看是否有变化（此时有变化）。

逐个刷新

如贵更新了配置文件之后，不希望每一个微服务都去刷新配置文件，那么可以进行如下的配置：

首先，给每一个config-client添加一个instance-id：

# 逐个刷新
eureka.instance.instance-id=${spring.application.name}:${server.port}

然后对config-client进行打包。

打包完成后，通过java -jar启动两个client实例：

java -jar config-client-0.0.1-SNAPSHOT.jar --server.port=8082
java -jar config-client-0.0.1-SNAPSHOT.jar --server.port=8083

修改配置文件并且提交到github之后，可以通过如下的方式只刷新某一个微服务，例如只刷新8082的微服务：

http://localhost:8081/actuator/bus-refresh/client01:8082

client01:8082表示服务的instance-id。
