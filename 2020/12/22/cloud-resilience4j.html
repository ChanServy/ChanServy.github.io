前言：Spring Cloud之Resilience4j。

Resilience4j

Resilience4j 是 Spring Cloud GreenWich 版推荐的容错解决方案，相比 Hystrix ， Resilience4j 专为 Java8 以及函数式编程而设计。由于奈飞的断更， Hystrix 将来有可能被 Resilience4j 取而代之。

Resilience4j 主要提供了如下功能：

  • 断路器
  • 限流
  • 基于信号量的隔离
  • 缓存
  • 限时
  • 请求重试

1、结合微服务

Retry、CircuitBreaker、RateLimiter。

1.1 Retry 请求重试

首先在父工程下建立 module 子项目，并命名为 CSY-Resilience4j。项目创建成功后，手动添加 Resilience4j 依赖。

<dependency>
    <groupId>org.springframework.cloud</groupId>
    <artifactId>spring-cloud-starter-netflix-eureka-client</artifactId>
</dependency>
<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot2</artifactId>
    <version>1.2.0</version>
    <exclusions>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-circuitbreaker</artifactId>
        </exclusion>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-ratelimiter</artifactId>
        </exclusion>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-bulkhead</artifactId>
        </exclusion>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-timelimiter</artifactId>
        </exclusion>
    </exclusions>
</dependency>

resilience4j-spring-boot2 中包含了 resilience4j 的所有功能，但是没有配置的功能无法使用，如要将之从依赖中剔除掉。

配置 yml 文件：

server:
  port: 20007

spring:
  application:
    name: CSY-resilience4j

eureka:
  client:
    service-url:
      defaultZone: http://localhost:20000/eureka/

resilience4j:
  retry:
    retry-aspect-order: 399 # 表示Retry的优先级
    backends:
      retryA:
        maxRetryAttempts: 5 # 重试次数
        waitDuration: 500 # 重试等待时间
        exponentialBackoffMultiplier: 1.1 # 间隔乘数
        retryExceptions:
          - java.lang.RuntimeException

/**
 * @Classname Resilience4jApplication
 * @Description TODO
 * @Date 2020/10/2 19:23
 * @Created by ChenSiYu
 */
@SpringBootApplication
@EnableDiscoveryClient
@RibbonClient(value = "CSY-PROVIDER")
public class Resilience4jApplication {
    public static void main(String[] args) {
        SpringApplication.run(Resilience4jApplication.class, args);
    }

    @Bean
    @LoadBalanced
    //给rest模板开启负载均衡,一定要加这个注解,不然的话会提示找不到主机
    RestTemplate restTemplate(){
        return new RestTemplate();
    }

}

/**
 * @Classname HelloController
 * @Description TODO
 * @Date 2020/10/2 21:47
 * @Created by ChenSiYu
 */
@RestController
public class HelloController {

    @Autowired
    HelloService helloService;

    @RequestMapping("/hi/{name}")
    public String hi(@PathVariable String name) {
        String servicePort = helloService.getServicePort(name);
        return servicePort;
    }
}

在远程调用服务的地方添加 @Retry 注解，实现请求重试。

/**
 * @Classname HelloServiceImpl
 * @Description TODO
 * @Date 2020/10/2 22:21
 * @Created by ChenSiYu
 */
@Service
@Retry(name = "retryA")//表示要使用的重试策略
public class HelloServiceImpl implements HelloService {


    @Autowired
    RestTemplate restTemplate;

    @Override
    public String getServicePort(String name) {
        return restTemplate.getForObject("http://CSY-PROVIDER/hi?name=" + name, String.class);
    }
}

1.2 CircuitBreaker

降级熔断策略。首先从依赖中排除 CircuitBreaker。

<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot2</artifactId>
    <version>1.2.0</version>
    <exclusions>
        <!--<exclusion>
                    <groupId>io.github.resilience4j</groupId>
                    <artifactId>resilience4j-circuitbreaker</artifactId>
                </exclusion>-->
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-ratelimiter</artifactId>
        </exclusion>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-bulkhead</artifactId>
        </exclusion>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-timelimiter</artifactId>
        </exclusion>
    </exclusions>
</dependency>

在 yml 中添加 CircuitBreaker 配置：

server:
  port: 20007

spring:
  application:
    name: CSY-resilience4j

eureka:
  client:
    service-url:
      defaultZone: http://localhost:20000/eureka/

resilience4j:
  retry:
    retry-aspect-order: 399 # 表示Retry的优先级
    backends:
      retryA:
        maxRetryAttempts: 5 # 重试次数
        waitDuration: 500 # 重试等待时间
        exponentialBackoffMultiplier: 1.1 # 间隔乘数
        retryExceptions:
          - java.lang.RuntimeException
  circuitbreaker:
    instances:
      cbA:
        ringBufferSizeInClosedState: 5
        ringBufferSizeInHalfOpenState: 3
        waitInterval: 5000
        recordExceptions:
          - org.springframework.web.client.HttpServerErrorException
    circuit-breaker-aspect-order: 398

配置完成后，用 @CircuitBreaker 标记相关的方法：

/**
 * @Classname HelloServiceImpl
 * @Description TODO
 * @Date 2020/10/2 22:21
 * @Created by ChenSiYu
 */
@Service
@Retry(name = "retryA")//表示要使用的重试策略
@CircuitBreaker(name = "cbA", fallbackMethod = "error")
public class HelloServiceImpl implements HelloService {


    @Autowired
    RestTemplate restTemplate;

    @Override
    public String getServicePort(String name) {
        return restTemplate.getForObject("http://CSY-PROVIDER/hi?name=" + name, String.class);
    }

    public String error(Throwable throwable){
        return "error";
    }
}

@CircuitBreaker 注解中的 name 属性是用来指定 CircuitBreaker 配置的，fallbackMethod 属性用来指定服务降级的方法，需要注意的是，服务降级方法中，要添加异常参数。

1.3 RateLimiter

RateLimiter 作为限流工具，大多数用在提供者服务，用来保护服务端接口。

首先在 CSY-provider 中添加 Resilience4j 依赖：

<dependency>
    <groupId>io.github.resilience4j</groupId>
    <artifactId>resilience4j-spring-boot2</artifactId>
    <version>1.2.0</version>
    <exclusions>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-circuitbreaker</artifactId>
        </exclusion>
        <!--<exclusion>
                    <groupId>io.github.resilience4j</groupId>
                    <artifactId>resilience4j-ratelimiter</artifactId>
                </exclusion>-->
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-bulkhead</artifactId>
        </exclusion>
        <exclusion>
            <groupId>io.github.resilience4j</groupId>
            <artifactId>resilience4j-timelimiter</artifactId>
        </exclusion>
    </exclusions>
</dependency>

修改 properties 文件：

#这里配置每秒钟处理一个请求
resilience4j.ratelimiter.limiters.rlA.limit-for-period=1
resilience4j.ratelimiter.limiters.rlA.limit-refresh-period=1s
#冷却时间
resilience4j.ratelimiter.limiters.rlA.timeout-duration=1s

为了查看效果，在 CSY-provider 的 HelloController 中打印每一个请求的时间：

/**
 * @Classname HelloController
 * @Description TODO
 * @Date 2020/9/1 17:31
 * @Created by ChenSiYu
 */
@RestController
public class HelloController {

    @Value("${server.port}")
    String port;

    @RateLimiter(name = "rlA")
    @RequestMapping("/hi")
    public String home(@RequestParam(value = "name", defaultValue = "erdochan") String name) {
        System.out.println(new Date());
        return "hi " + name + " ,i am from port:" + port;
    }

}

这里通过 @RateLimiter 来标记该接口限流。

配置完成后，重启 CSY-provider。

然后在 CSY-resilience4j （消费者服务）中模拟多个请求，查看限流效果：

/**
 * @Classname HelloController
 * @Description TODO
 * @Date 2020/10/2 21:47
 * @Created by ChenSiYu
 */
@RestController
public class HelloController {

    @Autowired
    HelloService helloService;

    @RequestMapping("/hi/{name}")
    public String hi(@PathVariable String name) {
        String servicePort = helloService.getServicePort(name);
        return servicePort;
    }

    /**
     * @Description 测试CSY-provider服务中模拟的限流
     * @Param [name]
     * @return java.lang.String
     * @Date 2020/10/2 23:16
     * @Author ChenSiYu
     */
    @RequestMapping("/rlA/{name}")
    public String hello(@PathVariable String name){
        for (int i = 0; i < 5; i++) {
            String servicePort = helloService.getServicePort(name);
        }
        //这里测试限流用,就不返回servicePort了
        return "success";
    }
}
