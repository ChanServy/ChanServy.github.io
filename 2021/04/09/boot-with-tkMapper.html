前言：Spring Boot之集成通用Mapper。

通用Mapper的接口，不同的接口会有不同的方法，适应不同的业务需求，因此为了满足大部分的业务需求，我们应该自定义Mapper接口，继承通用Mapper提供的各个接口。在项目中，如果使用通用Mapper，那么必然不止一个微服务会用到，因此将自定义的Mapper接口放到一个common模块会很方便其它服务的调用，其他服务模块直接引入common模块就可以。

当通用Mapper提供的方法不够满足业务需要并且通过Example拼接条件依然不够满足业务需要的时候，我们就可以像正常的mybatis一样实现，使用注解或者xml的方式，但是前提是xml的方式别忘了配置xml的路径。

package com.leyou.common.mapper;

import tk.mybatis.mapper.additional.idlist.IdListMapper;
import tk.mybatis.mapper.annotation.RegisterMapper;
import tk.mybatis.mapper.common.Mapper;
//import tk.mybatis.mapper.common.special.InsertListMapper;
import tk.mybatis.mapper.additional.insert.InsertListMapper;

/**
 * @Classname MyBaseMapper
 * @Description
 * 自定义的通用mapper接口,基本方法在Mapper<T>,
 * 通过id批量删除和查询在IdListMapper<T, Long>,
 * 批量新增在InsertListMapper<T>,
 * 批量新增有两个InsertListMapper<T>:
 * 1. import tk.mybatis.mapper.additional.insert.InsertListMapper;
 * 2. import tk.mybatis.mapper.common.special.InsertListMapper; 特别要求主键字段名必须写成"id"
 * @Date 2021/2/23 13:20
 * @Created by ChenSiYu
 */
@RegisterMapper
public interface MyBaseMapper<T> extends Mapper<T>, IdListMapper<T, Long>, InsertListMapper<T> {
}

JAVA

说明：

  • 表名默认使用类名,驼峰转下划线(只对大写字母进行处理),如UserInfo默认对应的表名为user_info
  • 表名可以使用@Table(name = “tableName”)进行指定,对不符合第一条默认规则的可以通过这种方式指定表名
  • 字段默认和@Column一样,都会作为表字段,表字段默认为Java对象的Field名字驼峰转下划线形式
  • 可以使用@Column(name = “fieldName”)指定不符合第3条规则的字段名
  • 使用@Transient注解可以忽略字段,添加该注解的字段不会作为表字段使用
  • 建议一定是有一个@Id注解作为主键的字段,可以有多个@Id注解的字段作为联合主键
  • 如果是MySQL的自增字段，加上@GeneratedValue(generator = “JDBC”)即可
  • @KeySql(useGeneratedKeys = true)是专属通用Mapper的注解,表示自增

public PageResult<SpuVo> querySpuByPageAndSort(Integer page, Integer rows, Boolean saleable, String key) {
    // 查询Spu
    // 分页,最多允许查100条
    PageHelper.startPage(page, Math.min(rows, 100));
    // 创建查询条件
    Example example = new Example(Spu.class);
    Example.Criteria criteria = example.createCriteria();
    // 是否过滤上下架
    if (saleable != null) {
        criteria.andEqualTo("saleable", saleable);
    }
    // 是否模糊查询 搜索字段过滤
    if (StringUtils.isNotBlank(key)) {
        criteria.andLike("title", "%" + key + "%");
    }
    // 默认排序
    example.setOrderByClause("last_update_time DESC");
    // 查询
    List<Spu> spus = this.spuMapper.selectByExample(example);
    // 判断
    if (CollectionUtils.isEmpty(spus)) {
        throw new LyException(ExceptionEnum.GOODS_NOT_FOUND);
    }

    // 将cname和bname加入 变成SpuVo的集合
    List<SpuVo> spuVos = loadCategoryAndBrandName(spus);

    // 解析最终的分页结果
    PageInfo<SpuVo> info = new PageInfo<>(spuVos);
    PageResult<SpuVo> result = new PageResult<>(info.getTotal(), spuVos);
    return result;
}

public PageResult<Brand> queryBrandByPage(Integer page, Integer rows, String sortBy, Boolean desc, String key) {
    //只要是单表的业务通用mapper基本上都能解决
    //page-helper其实就2步
    //1.分页
    PageHelper.startPage(page, rows);
    //条件过滤
    /* SELECT * FROM tb_brand WHERE `name` LIKE "%x%" OR letter == 'x' ORDER BY id DESC */
    Example example = new Example(Brand.class);
    if (StringUtils.isNotBlank(key)) {
        //过滤条件  WHERE `name` LIKE "%x%" OR letter == 'x'
        //通过tk.mybatis的example来设置条件
        example.createCriteria().orLike("name", "%" + key + "%").orEqualTo("letter", key.toUpperCase());
    }
    //排序 ORDER BY id DESC 其中 按照什么排序 和 升序还是降序 需要我们自己写子句指定
    if (StringUtils.isNotBlank(sortBy)) {
        String orderByClause = sortBy + " " + (desc ? "DESC" : "ASC");
        example.setOrderByClause(orderByClause);//Clause英文是子句的意思
    }
    //查询
    List<Brand> brands = brandMapper.selectByExample(example);
    if (CollectionUtils.isEmpty(brands)) {
        throw new LyException(ExceptionEnum.BRAND_NOT_FOUND);
    }
    //2.解析分页结果 方便封装成PageResult<Brand>返回
    PageInfo<Brand> info = new PageInfo<>(brands);
    PageResult<Brand> result = new PageResult<>(info.getTotal(), brands);
    return result;
}

public List<Sku> querySkuListBySpuId(Long id) {
    Sku record = new Sku();
    // 封装查询条件
    record.setSpuId(id);
    List<Sku> skus = this.skuMapper.select(record);//这个方法是按传入的对象属性非空的查
    if (CollectionUtils.isEmpty(skus)) {
        throw new LyException(ExceptionEnum.GOODS_NOT_FOUND);
    }
    //顺带着查库存
    for (Sku sku : skus) {
        Stock stock = this.stockMapper.selectByPrimaryKey(sku.getId());
        sku.setStock(stock.getStock());
    }
    return skus;
}

部分总结：

Select
    方法：List<T> select(T record);
    说明：根据实体中的属性值进行查询，查询条件使用等号

    方法：T selectByPrimaryKey(Object key);
    说明：根据主键字段进行查询，方法参数必须包含完整的主键属性，查询条件使用等号

    方法：List<T> selectAll();
    说明：查询全部结果，select(null)方法能达到同样的效果

    方法：T selectOne(T record);
    说明：根据实体中的属性进行查询，只能有一个返回值，有多个结果是抛出异常，查询条件使用等号

    方法：int selectCount(T record);
    说明：根据实体中的属性查询总数，查询条件使用等号

Insert
    方法：int insert(T record);
    说明：保存一个实体，null的属性也会保存，不会使用数据库默认值

    方法：int insertSelective(T record);
    说明：保存一个实体，null的属性不会保存，会使用数据库默认值

Update
    方法：int updateByPrimaryKey(T record);
    说明：根据主键更新实体全部字段，null值会被更新

    方法：int updateByPrimaryKeySelective(T record);
    说明：根据主键更新属性不为null的值

Delete
    方法：int delete(T record);
    说明：根据实体属性作为条件进行删除，查询条件使用等号

    方法：int deleteByPrimaryKey(Object key);
    说明：根据主键字段进行删除，方法参数必须包含完整的主键属性

    Example方法
    方法：List<T> selectByExample(Object example);
    说明：根据Example条件进行查询
    重点：这个查询支持通过Example类指定查询列，通过selectProperties方法指定查询列

    方法：int selectCountByExample(Object example);
    说明：根据Example条件进行查询总数

    方法：int updateByExample(@Param("record") T record, @Param("example") Object example);
    说明：根据Example条件更新实体record包含的全部属性，null值会被更新

    方法：int updateByExampleSelective(@Param("record") T record, @Param("example") Object example);
    说明：根据Example条件更新实体record包含的不是null的属性值

    方法：int deleteByExample(Object example);
    说明：根据Example条件删除数据
