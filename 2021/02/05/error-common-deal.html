异常统一处理方案

2021年2月5日
| 项目经验
| 阅读
文章目录

前言：最近在自己写项目，也是边写边学习边总结了。回想一下，之前在写项目的时候，当项目中出现异常的时候，总是潦草解决，脑海中没有一个统一的轮廓或者标准，所以想来总结一下。

一、为什么要统一异常处理

我们在项目中，往往需要查询什么数据返回给前端，或者将前端传来的数据插入到数据库等等，比如下面的例子中的代码，保存商品对象到数据库中时，我需要校验价格，价格如果为空，就抛出异常不进行插入。注：下面的代码只是我为了测试随便写的一个简易的例子。

/**
 * @Classname ItemController
 * @Description
 * ---@ResponseBody就是将返回的对象序列化成(一般是json)放到响应体中,也就是能够处理方法结果值作为http响应体
 * ---@ResponseStatus：加在方法上、返回自定义http状态码
 * ---ResponseEntity标识整个http相应：状态码、头部信息以及相应体内容。因此我们可以使用其对http响应实现完整配置。
 * @Date 2021/2/2 13:24
 * @Created by ChenSiYu
 */
@RestController
@RequestMapping("/item")
public class ItemController {

    @Autowired
    private ItemService itemService;

    @PostMapping("/saveItem")
    public ResponseEntity<Item> saveItem(Item item) {
        //校验价格
        if (item.getPrice() == null) {
            // 新增失败返回400
            // return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(null);
            // throw new RuntimeException("价格不能为空!");
            throw new LyException(ExceptionEnum.PRICE_CANNOT_BE_NULL);
        }
        Item saveItem = itemService.saveItem(item);
        // 新增成功返回201状态码 并且显示新增对象
        return ResponseEntity.status(HttpStatus.CREATED).body(saveItem);
    }
}

@Service
public class ItemServiceImpl implements ItemService {
    @Override
    public Item saveItem(Item item) {
        // 商品新增
        int id = new Random().nextInt(100);
        item.setId(id);
        return item;
    }
}

代码中可以看出，我最开始的做法是：return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(null);，直接通过 ResponseEntity 返回状态码和错误信息，但是我插入数据后显示了新增的对象，既然有返回值，那么ResponseEntity 的泛型必须是这个返回的对象的类型。不能在 body() 里面写 String 类型的错误信息。想显示错误信息，然后我又 throw new RuntimeException("价格不能为空!"); 。然后新建了个通用异常处理器。

因为在一个微服务项目中，会有多个微服务，既然是异常处理模块，那么每个服务都会用到的，因此我把统一异常处理写在一个公用的模块，在这里要注意包结构。

二、定义通用异常处理器

看下面的代码，@ExceptionHandler(RuntimeException.class) 和 return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(e.getMessage());，现在 String 错误信息是可以显示了。

/**
 * @Classname CommonExceptionHandler
 * @Description controller通用异常处理器
 * 这个通用异常处理器想生效全靠 @ControllerAdvice 注解 这个注解在com.leyou.common
 * 这里在ly-item-service中使用到这个异常处理器 我们的ly-item-service模块中的启动类在com.leyou
 * 因此启动类在扫描的时候会扫描com.leyou 自然可以扫到 @ControllerAdvice
 * @Date 2021/2/2 21:31
 * @Created by ChenSiYu
 */
@ControllerAdvice//拦截所有controller 有异常则执行handleException
public class CommonExceptionHandler {

//    @ExceptionHandler(RuntimeException.class)//拦截RuntimeException类型的异常
    @ExceptionHandler(LyException.class)
    public ResponseEntity<ExceptionResult> handleException(LyException e){
        // 这里的状态码写死了 但是因为这是通用异常处理器 所以不一定是这个BAD_REQUEST这个
        // 想在ItemController中RuntimeException中再传一个状态码过来 但发现RuntimeException不能传
        // 因此自定义异常类继承RuntimeException来实现 并且事先在枚举中统一状态码和异常信息
        // return ResponseEntity.status(HttpStatus.BAD_REQUEST).body(e.getMessage());
        ExceptionEnum exceptionEnum = e.getExceptionEnum();
        return ResponseEntity.status(exceptionEnum.getCode()).body(new ExceptionResult(exceptionEnum));
    }

}

虽然现在 String 错误信息是可以显示了，但是这种方式返回的状态码是固定的，我们这是个通用的异常处理模块，这样肯定是不行的。想在前面的 controller 中抛 RuntimeException 异常的时候带上状态码，可是 RuntimeException 并没有相应的构造函数。因此，我想自己造一个异常类继承 RuntimeException 来实现，并且事先在枚举中统一状态码和异常信息。

三、定义异常类型枚举

/**
 * @Classname ExceptionEnum
 * @Description 枚举是指具有固定实例个数的类
 * getter noArgsConstructor allArgsConstructor
 * @Date 2021/2/2 21:54
 * @Created by ChenSiYu
 */
public enum ExceptionEnum {

    // 写在最前 这些随着项目的开发迭代可能会日益变多
    PRICE_CANNOT_BE_NULL(400, "价格不能为空!"),
    CATEGORY_NOT_FOUND(404, "商品分类没查到!"),
    BRAND_NOT_FOUND(404, "品牌没查到!"),
    BRAND_SAVE_ERROR(500, "新增品牌失败!"),
    UPLOAD_FILE_ERROR(500, "文件上传失败!"),
    INVALID_FILE_TYPE(400,"无效的文件类型!"),
    ;
    private int code;
    private String msg;

    ExceptionEnum() {
    }

    ExceptionEnum(int code, String msg) {
        this.code = code;
        this.msg = msg;
    }

    public int getCode() {
        return code;
    }

    public String getMsg() {
        return msg;
    }


}

四、自定义异常类

引入自定义的 ExceptionEnum ，并完成相应的所有构造。

/**
 * @Classname LyException
 * @Description 自定义异常 getter noArgsConstructor allArgsConstructor
 * @Date 2021/2/2 21:53
 * @Created by ChenSiYu
 */
public class LyException extends RuntimeException {

    private ExceptionEnum exceptionEnum;

    public LyException(ExceptionEnum exceptionEnum) {
        this.exceptionEnum = exceptionEnum;
    }

    public LyException(String message, ExceptionEnum exceptionEnum) {
        super(message);
        this.exceptionEnum = exceptionEnum;
    }

    public LyException(String message, Throwable cause, ExceptionEnum exceptionEnum) {
        super(message, cause);
        this.exceptionEnum = exceptionEnum;
    }

    public LyException(Throwable cause, ExceptionEnum exceptionEnum) {
        super(cause);
        this.exceptionEnum = exceptionEnum;
    }

    public LyException(String message, Throwable cause, boolean enableSuppression, boolean writableStackTrace, ExceptionEnum exceptionEnum) {
        super(message, cause, enableSuppression, writableStackTrace);
        this.exceptionEnum = exceptionEnum;
    }

    public LyException() {
    }

    public LyException(String message) {
        super(message);
    }

    public LyException(String message, Throwable cause) {
        super(message, cause);
    }

    public LyException(Throwable cause) {
        super(cause);
    }

    public LyException(String message, Throwable cause, boolean enableSuppression, boolean writableStackTrace) {
        super(message, cause, enableSuppression, writableStackTrace);
    }

    public ExceptionEnum getExceptionEnum() {
        return exceptionEnum;
    }
}

到此，在通用异常处理器中 @ExceptionHandler(LyException.class) ，return ResponseEntity.status(exceptionEnum.getCode()).body(exceptionEnum.getMsg()); 所有问题迎刃而解！但是有个问题，这里显示的错误信息只有一段字符串，显得太单调了。。。所以我又封装了一个异常结果类。

五、封装异常结果类

/**
 * @Classname ExceptionResult
 * @Description 因为之前返回的错误信息只有一个字符串太简陋了 因此我们自定义一个返回结果对象
 * @Date 2021/2/2 22:19
 * @Created by ChenSiYu
 */
public class ExceptionResult {
    private int status;
    private String message;
    private String timestamp;

    // 怎么构建这个对象呢? 通过刚刚的枚举类来构建
    public ExceptionResult(ExceptionEnum exceptionEnum) {
        this.status = exceptionEnum.getCode();
        this.message = exceptionEnum.getMsg();
        this.timestamp = DateUtils.getDateToString(System.currentTimeMillis());
    }

    public int getStatus() {
        return status;
    }

    public void setStatus(int status) {
        this.status = status;
    }

    public String getMessage() {
        return message;
    }

    public void setMessage(String message) {
        this.message = message;
    }

    public String getTimestamp() {
        return timestamp;
    }

    public void setTimestamp(String timestamp) {
        this.timestamp = timestamp;
    }
}

在通用异常处理器中，return ResponseEntity.status(exceptionEnum.getCode()).body(new ExceptionResult(exceptionEnum)); ，在 controller 中，这样抛异常：throw new LyException(ExceptionEnum.PRICE_CANNOT_BE_NULL); 。
